# Web AI Assistant 확장앱 소개서

> 버전 기준: `1.7.2`
> 기준 범위: 현재 구현 + 진행 중인 기획 문서 반영

---

## 1. 제품 개요

Web AI Assistant는 브라우저 안에서 AI와 대화하면서 웹페이지를 읽고, 질문하고, 입력하고, 자동화할 수 있도록 만든 Chrome 확장앱입니다.

이 확장앱은 단순한 채팅 도구가 아니라 다음 세 가지를 한 제품 안에 담는 것을 목표로 합니다.

- **브라우저 AI 채팅 도구**
- **웹페이지 보조 입력 및 자동화 코파일럿**
- **사용자 맞춤형 사이트 확장 기능을 만드는 기반**

즉, 사용자는 웹페이지를 보면서 바로 AI에게 질문할 수 있고, 필요한 경우 AI가 브라우저를 직접 조작하도록 요청할 수 있으며, 더 나아가 반복 작업을 사이트별 스크립트로 발전시킬 수 있습니다.

---

## 2. 핵심 가치

### 2.1 페이지 문맥 기반 AI

일반적인 웹 AI 확장 기능이 단순 채팅에 머무는 것과 달리, Web AI Assistant는 현재 보고 있는 페이지와 선택한 텍스트, 입력 필드, 스크린샷, 페이지 구조를 함께 활용합니다.

이를 통해 다음과 같은 작업이 가능합니다.

- 현재 페이지 내용 요약
- 선택 문장 번역 및 설명
- 페이지 내용을 바탕으로 질의응답
- 입력 필드 분석 후 자연어 기반 자동입력

### 2.2 브라우저 액션 자동화

사용자는 자연어로 요청하고, AI는 필요 시 브라우저 액션 도구를 사용해 실제 작업을 수행합니다.

예:
- 페이지 이동
- 버튼 클릭
- 폼 입력
- 목록/댓글 추출
- 탭 그룹 정리
- 방문 기록 검색
- 최근 닫은 탭 복원

### 2.3 스크립트 기반 확장성

단순히 한 번 작업을 대신해주는 수준을 넘어서, 페이지에서 실행한 스크립트를 추적하고, 영구 스크립트로 발전시키고, 향후 Studio 형태로 관리할 수 있는 방향으로 확장 중입니다.

이 구조는 “AI 자동화”를 “사용자 맞춤 확장앱 자산”으로 바꾸는 기반이 됩니다.

## 3. 주요 기능

### 3.1 AI 채팅

- Chrome 사이드 패널 기반 AI 채팅
- 스트리밍 응답
- 페이지 문맥 기반 질문
- 대화형 후속 작업

### 3.2 텍스트 선택 기반 기능

- 우클릭 메뉴를 통한 선택 텍스트 AI 처리
- 선택 문장과 주변 문맥을 함께 전달
- 선택 텍스트 질문, 설명, 가공

진행 중인 기획:
- 선택 직후 작은 아이콘이 뜨는 **Floating Helper**
- 빠른 번역, 질문, 요약 진입

### 3.3 페이지 도우미

- 현재 페이지 요약
- 페이지 기반 질문
- 입력 필드 분석
- 페이지 상황에 맞는 보조 작성

### 3.4 문서 및 폼 자동입력

- 일반 입력 필드 자동입력
- 리치 에디터 대응
- 그룹웨어 문서 작성 보조
- 작업요청(Inwork) 작성 보조
- 메일 작성 자동화 보강 진행 중

### 3.5 Action Tools 브라우저 에이전트

브라우저를 직접 조작하는 도구 체계:

- `GOTO`, `CLICK`, `FILL`, `PRESS`
- `ANALYZE_PAGE_STRUCTURE`, `EXTRACT_LIST_ITEMS`, `EXTRACT_ELEMENTS`
- `GROUP_TABS`, `UNGROUP_TO_WINDOWS`
- `SEARCH_HISTORY`, `RESTORE_TAB`
- `EXECUTE_SCRIPT`, `REGISTER_SCRIPT`

이 기능을 통해 사용자는 단순 답변이 아니라 실제 작업 실행을 요청할 수 있습니다.

### 3.6 이미지 및 화면 기반 기능

- 이미지 드래그 앤 드롭
- 스크린샷 캡처 후 질문
- Vision 모델 연동 시 이미지 분석

### 3.7 텍스트 개선

- 입력 필드에서 직접 문장 다듬기
- 초안 보정
- 문체 개선 및 정리

## 5. 어떤 사용자에게 적합한가

### 5.1 일반 사용자

- 웹페이지 내용을 빠르게 이해하고 싶은 사용자
- 선택 문장을 바로 번역/질문/요약하고 싶은 사용자
- 이메일, 문서, 요청서 작성을 보조받고 싶은 사용자

### 5.2 업무 사용자

- 그룹웨어, 포털, 내부 시스템 입력 작업이 많은 사용자
- 반복 탭 정리, 기록 검색, 페이지 탐색 자동화가 필요한 사용자
- 페이지 내용을 요약하고 정리해야 하는 사용자

### 5.3 파워 유저 및 운영자

- 페이지 스크립트 자동화가 필요한 사용자
- 사이트별 맞춤 동작을 만들고 싶은 사용자
- 향후 Studio 기능으로 자신의 확장 로직을 관리하고 싶은 사용자

---

## 6. 차별화 포인트

### 6.1 채팅과 브라우저 액션이 한 제품 안에 있음

많은 AI 확장 기능이 “질문-답변”에 머무는 반면, Web AI Assistant는 필요할 때 직접 브라우저를 조작할 수 있는 도구 체계를 갖습니다.

### 6.2 단발성 자동화에서 재사용 가능한 자산으로 확장

현재 진행 중인 script continuity와 Extension App Studio 방향은, AI가 한 번 실행한 작업을 다시 조회하고, 영구화하고, 편집 가능한 자산으로 바꾸는 데 초점을 둡니다.

### 6.3 페이지 문맥과 실제 UI를 함께 다룸

텍스트만 다루는 AI가 아니라, 페이지 구조, 입력 필드, 탭, 스크립트, 선택 문맥을 함께 다루는 것이 강점입니다.

---

## 7. 안전 및 운영 원칙

이 확장앱은 브라우저 자동화를 포함하므로 안전 경계가 중요합니다.

현재 제품 방향의 핵심 원칙:

- 위험한 작업은 승인 기반으로 제어
- 영구 스크립트는 명시적 등록 흐름 사용
- 세션 스크립트는 추적 가능한 형태로 관리 강화
- 민감한 사이트나 최종 제출 단계는 보수적으로 처리

특히 정부기관, 금융, 인증, 전송, 제출 같은 흐름은 완전 자동화보다 **보조 입력 + 사용자 확인** 방식이 더 적합합니다.

## 8. 현재 진행 중인 확장 방향

### 8.1 Floating Helper

선택한 문장 옆에 작은 helper 아이콘을 띄워서:

- 번역
- 질문하기
- 요약

같은 기능으로 즉시 이어지게 하는 UX를 준비 중입니다.

### 8.2 Session Script Continuity

`EXECUTE_SCRIPT` 실행 이력을 세션 단위로 관리하고:

- 무엇을 적용했는지 조회
- 방금 한 작업 되돌리기
- 세션 작업을 영구 스크립트로 승격

할 수 있도록 보강 중입니다.

### 8.3 Extension App Studio

장기적으로는 사용자가 AI와 대화하면서:

- 사이트별 스크립트를 조회하고
- 편집하고
- 버전 관리하고
- 라이브러리처럼 관리하는

Studio 기능으로 확장하는 방향을 갖고 있습니다.

- “방금 적용한 스크립트를 보여줘”

## 10. 정리

Web AI Assistant는 단순한 브라우저 챗봇이 아니라,

- 웹페이지를 이해하고
- 사용자의 현재 작업 문맥을 활용하며
- 필요 시 브라우저를 직접 조작하고
- 나아가 사이트별 자동화를 자산화하려는

AI 기반 브라우저 생산성 플랫폼입니다.

현재는 채팅, 페이지 도우미, 브라우저 액션, 스크립트 실행 기반이 구현되어 있고, 앞으로는 Floating Helper와 Studio를 통해 더 빠르고 더 재사용 가능한 사용자 경험으로 발전시키는 것이 목표입니다.
